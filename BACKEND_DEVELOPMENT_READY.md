## üö® Common Pitfalls to Avoid

### 1. **Don't Skip RLS Policies**
‚ùå **Wrong:** Create tables without RLS
‚úÖ **Right:** Enable RLS on EVERY table immediately

### 2. **Don't Use Service Role in Client**
‚ùå **Wrong:** Put SUPABASE_SERVICE_ROLE_KEY in client code
‚úÖ **Right:** Only use SUPABASE_ANON_KEY in client

### 3. **Don't Hardcode Organization IDs**
‚ùå **Wrong:** `WHERE organization_id = 'abc123'`
‚úÖ **Right:** `WHERE organization_id = getCurrentOrgId()`

### 4. **Don't Skip Type Generation**
‚ùå **Wrong:** Manually define types
‚úÖ **Right:** Generate types after every migration

### 5. **Don't Test RLS in Code**
‚ùå **Wrong:** `if (user.orgId === resource.orgId) { ... }`
‚úÖ **Right:** Let RLS handle it automatically

### 6. **Don't Skip Migration Rollbacks**
‚ùå **Wrong:** Only write UP migrations
‚úÖ **Right:** Document DOWN migration for every UP

### 7. **Don't Push Directly to Main**
‚ùå **Wrong:** `git push origin main`
‚úÖ **Right:** Use PRs from `feature/supabase-backend-integration`

### 8. **Don't Skip Testing**
‚ùå **Wrong:** "I'll test later"
‚úÖ **Right:** Test after EVERY phase
